1 Binary Function Review

Until this week, we have implemented binary functions as unary functions from pairs of values to results. The simplest example of this approach is found in our first implementation, andp, of the Boolean and function. As the type declaration indicates, andp takes a single argument comprising a pair of Boolean values, represented as a value of type pair bool bool. It then computes and returns the desired result. 

andp: pair bool bool -> bool
andp (mkPair true true ) = true
andp _ = false

2 Higher-Order Functions

This week we saw that we can view binary functions in a new way: as unary functions (taking one argument) that return functions, which in turn take another argument and compute and return a final result. A simple example is found in our second implementation, pand, of the Boolean and function. The type of pand explains that the function takes one value of type bool and then returns a unary function of type \(bool \to bool\). The function definition rules explain which unary function is returned for each value of the first argument. We already have named functions for all four possible unary functions from bool to bool, so we just return the right ones by name. 

pand: bool -> (bool -> bool)
pand true = id_bool
pand false = constFalse

We have thus met our first higher order function. We use this term to refer to any function that returns a function as a result (as in this example), or that takes a function as an argument, or both.

Given this definition of pand, we can now write function application expressions such as this: (pand true) false. The expression within parentheses evalutes to the function, id_bool. It is then applied to false, returning false as the final result. Because function application is left associative, we can omit the parentheses and simply write pand true false.

Defining Idris functions in this manner simplifies our code. First, we no longer need to use the pair type constructor in the type of our implementation of Boolean and. Second, we no longer need to package up two arguments into a single pair when we want to apply a Boolean and function. Third, the rules for computing values of the function no longer need to destructure expressions using pair. Every aspect of the code is simpler.

3 Partial Evaluation

So how should we think about the expression pand true in the code just introduced? Clearly it is the function id_bool. But an even better way to understand it, in this context, is as what we will call a partially evaluated application of a Boolean and function. That is, it's sort of like an and function with the first argument value already baked in. The binary function has been partially evaluated yielding a unary function that can now convert a second argument into a final result.

Now we're ready to see an even better implementation of our binary and function. This is implementation is based on the fact that Idris and other languages like it automatically partially evaluate function applications!

The type of our next function is the same as that of the the high-order function that we just implemented: \(bool \to bool \to bool\). The difference is that rather than writing rules that pattern match on just one argument and that return a function explicitly, we write we pattern match on both arguments. Our rules explain how to reduce both arguments to a final result. What's remarkable, however, is that if we then apply such a function to just one argument, Idris does the partial evaluation for us!

Here's the code. We now call our function and. 

and: bool -> bool -> bool 
and true true = true
and _ _ = false

Given this definition, we can continue to write function application expressions such as and true false. This expression does not match the first rule, but does match the second rule, which returns false.

Now here's the really interesting question: Does the expression, and true—the application of a binary function to one argument—have a meaning, and if so, what? The type of the and function explains that the result should be of type \(bool \to bool\). So does it work? Can we write the following code, and if so, what does it mean? 

uf: bool -> bool
uf = (and true)

In fact, it works and produces exactly the result expected: a partially evaluated result in the form of a unary function that is equivalent to our id_bool function. Idris and similar languages have the remarkable capability to partially evaluate multi-argument function definitions. We do not have to actually write the partially evaluating functions ourselves, as we did above.

One can now see how Idris evaluates function application expressions such as and true false. First it left associates terms, grouping the expression like this: (and true) false. Then it evaluates the (and true) function application expression, yielding a partially evaluated function which it then applies to false to compute the final result.

You can even capture and store the partially evaluated function, which is exacty what we did with the code above. Give it a try in your REPL. You can apply uf to single Boolean values to obtain final results.

We have now reached a milestone in this course: the standard pattern for implementing logical and mathematical functions that take multiple arguments as function definitions in Idris and in similar languages.

4 The Concepts Generalize

As you would hope, the same approach works not ony for bool but for multi-argument functions involving any types. Here for example is the code we wrote a while ago for natural number addition. 

||| given a pair of natural numbers, return its sum
addp: pair nat nat -> nat
addp (mkPair O m) = m
addp (mkPair (S n) m) = S (addp (mkPair n m))

Here is code that also implements natural number addition, but written in the style we just developed. The code is not only much simplified in comparison to the code above, but also supports partial evaluation with respect to just one of the Boolean arguments. The code above does not. 

||| given two natural numbers, return their sum
add: nat -> nat -> nat
add O m = m
add (S n) m = S (add n m)

Suppose I want a function to add \(2\) points to any given homework score. I could produce such a function by applying add to \(2\)! The result is a partially evaluated unary function that takes any score and return that score plus \(2\). 

addTwo: nat -> nat
addTwo = add (S (S O))

result: nat
result = addTwo (S (S (S O))) --yields five!
